# Alert

The Alert component is a modal dialog used to display important messages and request user confirmations. It's built on Headless UI for accessibility and transitions.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `ReactNode` | Required | The alert's title |
| `description` | `ReactNode` | Required | The main content of the alert |
| `show` | `boolean` | Required | Controls the alert's visibility |
| `isDestructive` | `boolean` | `false` | If true, highlights the alert as a destructive action |
| `isPerformingAction` | `boolean` | `false` | Shows a loading state while performing the action |
| `confirmText` | `string` | `'Confirm'` | Text for the confirmation button |
| `children` | `ReactNode` | - | Additional content to display in the alert |
| `onConfirm` | `() => void` | - | Callback executed on confirmation |
| `onClose` | `() => void` | Required | Callback executed when closing |

## Usage Examples

### Basic Alert
```tsx
import { Alert } from 'mycrumbs-uikit';

function Example() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <Alert
      show={isOpen}
      title="Confirm Action"
      description="Are you sure you want to proceed with this action?"
      onConfirm={() => {
        // handle confirmation
        setIsOpen(false);
      }}
      onClose={() => setIsOpen(false)}
    />
  );
}
```

### Destructive Alert
```tsx
<Alert
  show={isOpen}
  title="Delete Account"
  description="This action cannot be undone. All data will be permanently deleted."
  isDestructive
  confirmText="Delete"
  onConfirm={handleDelete}
  onClose={() => setIsOpen(false)}
/>
```

### Alert with Custom Content
```tsx
<Alert
  show={isOpen}
  title="Additional Information"
  description="Please review the following details:"
  onClose={() => setIsOpen(false)}
>
  <div className="mt-4">
    <ul className="list-disc pl-4">
      <li>Detail 1</li>
      <li>Detail 2</li>
    </ul>
  </div>
</Alert>
```

## Accessibility

- Uses appropriate ARIA roles for modal dialog
- Handles focus trap correctly
- Supports ESC key dismissal
- Provides smooth transitions for better user experience

## Technical Notes

- Built on [@headlessui/react](https://headlessui.dev/)
- Supports customization through children
- Automatically handles enter/leave animations
- Responsive and adaptable to different use cases

## Best Practices

1. Use clear and concise titles
2. Provide descriptive content in the description
3. Use destructive styling only for irreversible actions
4. Keep confirmation text actionable and specific
5. Consider using loading states for async operations

## Implementation Details

The Alert component is built using Headless UI's Dialog component and includes:
- Transition animations for smooth entry/exit
- Backdrop click handling
- Keyboard interaction support
- Focus management
- Screen reader compatibility
